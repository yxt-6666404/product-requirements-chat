# 产品需求对话 Agent（Product Requirements Chat）

## 文件夹说明

本文件夹包含产品需求对话 Agent 的完整配置与资源，用于将模糊的产品想法或零散材料通过对话澄清为可执行的需求产物。

### 主要功能

- **需求澄清与结构化**：通过对话将模糊想法转化为结构化的需求文档
- **多模式工作流**：支持探索（发散）、收敛（决策）、落文档（交付）三种工作模式
- **智能工具路由**：根据用户意图自动选择合适的 Skill/MCP 工具，提升产出质量
- **模板化输出**：提供 PRD、用户故事、验收标准、范围与优先级、风险/依赖等多种模板
- **范围管理**：持续维护 In scope / Out of scope / 待确认 的范围边界

### 文件夹结构

```
product-requirements-chat/
├── agent.md             # System Prompt 与核心配置
├── dialog-flow.md       # 对话流程与模式切换规则
├── examples.md          # 使用示例
├── output-templates.md  # 结构化输出模板
├── skill-tool-map.md    # Skill/MCP 工具映射表
└── skill/
    └── SKILL.md         # Skill 定义文件
```

## 如何使用该产品需求对话 Agent

### 方法一：以 Trae 为例使用 Agent

1. **打开 Trae 的 solo 模式**
2. **进入 /spec 模式**
3. **输入以下指令**：
   ```
   你是有10多年经验的产品需求对话师，读取 product-requirements-chat 文件夹（可以自己复制文件路径询问）下的文件，成为专业的产品需求分析 agent
   ```
4. **开始对话**：直接描述你的产品想法或提供已有材料，Agent 会引导你完成需求澄清与结构化

### 方法二：使用 Skill

1. **在 Trae 的 skills 目录下创建 product-requirements-chat 文件夹**
2. **将本文件夹中的 skill/SKILL.md 文件复制到该目录**
3. **在对话中使用**：当你需要通过对话澄清需求时，Trae 会自动识别并调用该 Skill

## 核心能力

- **快速启动**：只需一句话描述，Agent 会自动补齐关键维度
- **材料处理**：支持处理 PRD、原型、截图、链接、竞品名称等多种材料
- **灵活产出**：可根据需要输出 PRD、用户故事、验收标准、范围与优先级表、竞品调研摘要等
- **智能工具使用**：优先使用专门的 Skill/MCP 工具，不可用时自动降级为纯对话+模板输出
- **持续范围管理**：每轮对话都维护范围边界，确保需求清晰可控

## 工作模式

- **探索模式**：当信息不足时，覆盖 Who/When&Where/What/Why/Success/Constraints/Reference 等维度
- **收敛模式**：当信息基本齐备时，对齐成功指标与边界，输出优先级与验收标准
- **落文档模式**：当需要交付时，先给提纲再展开，生成结构化的需求文档

## 输出模板

本 Agent 提供多种结构化输出模板，包括：
- 每轮最小输出（当前理解 + 追问 + 范围边界）
- 需求摘要（PRD 快速版）
- 用户故事 + 验收标准（Given/When/Then）
- 范围与优先级（维护表）
- 风险 / 依赖 / 决策记录

## 注意事项

- **工具降级**：当推荐的 Skill/MCP 工具不可用时，Agent 会自动降级为纯对话+模板输出
- **范围管理**：每轮对话都会更新范围边界，确保需求清晰可控
- **证据引用**：涉及外部信息时，会提供可验证的来源链接
- **隐私保护**：不收集与记录不必要的敏感信息

## 快速开始

1. 选择上述任一方法配置 Agent
2. 在对话框中描述你的产品想法或提供已有材料
3. 跟随 Agent 的引导，逐步澄清需求并生成结构化产出
4. 根据需要，要求 Agent 输出特定格式的文档或表格
