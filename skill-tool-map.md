# Skill / MCP 工具映射表（产品需求对话 Agent）

## 可用 skills（本机 skills 目录可见；不可用则降级为纯对话/模板输出）

| Skill | 适用意图 | 主要产出 |
|---|---|---|
| brainstorming | 需求发散、问题空间探索、假设枚举、快速对齐方向 | 问题树、方案备选、风险清单 |
| doc-coauthoring | PRD/方案/说明文共创（提纲→细化→校验） | PRD/方案文档、章节提纲、校验清单 |
| pricing-strategy | 定价模型、套餐、试用/付费墙、价值度量与验证 | 定价方案备选、实验设计、指标与风险 |
| content-strategy | 内容规划、信息架构、内容节奏与栏目 | 内容结构、内容矩阵、渠道节奏 |
| copywriting | 具体文案撰写与润色（落地页/按钮/邮件等） | 文案版本、语气与可读性修订 |
| marketing-psychology | 转化机制、用户决策、行为引导与实验假设 | 转化策略、A/B 假设、触发机制设计 |
| agent-browser | 需要交互验证、截图取证、走通关键流程 | 关键路径截图、交互记录、证据引用 |
| executing-plans | 里程碑、拆分、排期与执行节奏 | 里程碑计划、任务拆分、推进节奏 |
| request-analyzer（可选） | 识别意图、拆解诉求、建议下一步 | 意图摘要、关键缺口、推荐路径 |
| 纯对话/模板输出 | skills 不可用、或用户只要快速可复制结果 | 按模板输出的 PRD/表格/清单（Markdown） |

## 可用 MCP 工具（当前运行环境可见）

| 工具 | 适用意图 | 主要产出 |
|---|---|---|
| WebSearch | 快速获取公开资料、定位权威来源、收集竞品/参考链接 | 来源列表（URL）、要点摘要 |
| WebFetch（若可用） | 抓取单个网页内容用于精读与引用 | 网页内容摘录、引用段落、结构化要点 |
| Playwright | 浏览器自动化（导航/点击/表单/截图/提取HTML/必要时HTTP请求）用于交互验证与取证 | 关键路径截图、页面HTML/文本摘录、交互取证记录 |
| Excel | 表格沉淀与交付（读写sheet/建表/批量写入/格式化/截图）用于结构化记录与可交付导出 | 工作簿/表格内容、格式化后的交付表、表格截图 |

## 降级路径约定（当推荐 skill/工具不可用）

- 需求梳理/PRD/用户故事/验收标准：使用 output-templates.md 的模板，以纯对话方式输出 Markdown。
- 竞品/公开资料调研：无法使用 WebSearch 时，改为“需要用户提供链接/材料”并基于材料做摘要与对比；找不到 WebFetch 时，用 Playwright 打开页面并提取可见文本/HTML 作为引用依据。
- 交互路径取证：无法使用 Playwright/浏览器能力时，请用户提供截图/录屏/具体页面链接，并基于证据复盘流程。
- 表格沉淀/导出：无法使用 Excel/表格工具时，输出可复制的 Markdown 表格与字段规范（必要时附 CSV 列表）。

## 意图 → 推荐 skill/工具 → 产出（选择表）

| 用户意图（你听到的诉求） | 推荐 skill/工具（按优先顺序） | 产出（交付给用户） |
|---|---|---|
| 只有一句话想法/目标，要先把需求“聊清楚” | request-analyzer（可选） → brainstorming（如可用） → 纯对话/模板输出 | 当前理解摘要、追问清单、待确认项列表 |
| 已有一些材料（PRD/原型/链接），要基于材料继续迭代 | doc-coauthoring（如可用） → WebFetch（若可用）/Playwright（提取页面文本） → 纯对话/模板输出 | 变更点复述、影响面分析（范围/依赖/风险）、新增追问 |
| 要把讨论内容整理成 PRD/需求草案 | doc-coauthoring（如可用） → 纯对话/模板输出 | PRD 草案（Why/Who/Scope/功能清单/验收标准/风险与依赖/下一步） |
| 要输出用户故事与验收标准（Given/When/Then） | doc-coauthoring（如可用） → 纯对话/模板输出 | 用户故事列表、验收标准（可测/可验证） |
| 要做功能清单与优先级排序（P0/P1/P2 或 MoSCoW） | doc-coauthoring（如可用） → 纯对话/模板输出（Markdown 表格） | 优先级表、范围边界（In/Out）、里程碑建议 |
| 需求涉及定价/套餐/试用/付费墙 | pricing-strategy（如可用） → WebSearch（竞品定价） | 定价模型备选、包装建议、验证/实验方案、风险点 |
| 需求涉及内容规划（栏目/信息架构/节奏） | content-strategy（如可用） → 纯对话/模板输出 | 内容结构、信息架构、内容节奏与衡量指标 |
| 需求涉及对外文案（落地页/按钮/邮件/社媒） | copywriting（如可用） → content-strategy（如可用） → 纯对话/模板输出 | 文案初稿/多版本、语气与可读性优化点 |
| 需求涉及转化机制/行为引导/增长实验 | marketing-psychology（如可用） → WebSearch（参考案例） | 转化策略、实验假设、指标定义、反作弊/副作用 |
| 要做竞品/参考调研（只要公开信息，不需交互） | WebSearch → WebFetch（若可用）/Playwright（提取页面文本） | 来源列表、结论摘要、可借鉴点与风险点 |
| 要验证某网站/产品的真实交互路径并要截图证据 | Playwright（如可用） → agent-browser（如可用） → WebFetch（若可用，用于补充页面文字） | 关键路径截图与注释要点、流程复盘、差异与启发 |
| 要把需求清单变成可维护的表格/导出（CSV/XLSX） | Excel（如可用） → 纯对话/模板输出（Markdown 表格 + CSV 字段规范） | 需求表（字段：模块/描述/优先级/状态/负责人/验收标准）、可复制表格 |
| 要做范围管理（确认/待确认/不做）并持续更新 | doc-coauthoring（如可用） → 纯对话/模板输出 | 范围边界表、待确认问题清单、决策记录 |
| 要做风险与依赖盘点（技术/资源/合规/数据） | doc-coauthoring（如可用） → WebSearch（合规/行业要求） | 风险清单、依赖项列表、缓解措施与决策点 |
| 要制定下一步执行计划（里程碑/切分/投入） | executing-plans（如可用） → 纯对话/模板输出 | 里程碑计划、任务拆分、需要用户补充的信息清单 |
